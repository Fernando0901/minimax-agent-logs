## Failed Tests — Detail

### T08 — Image analysis (understand_image)
**Expected:** Returns text description of the Wikimedia PNG image
**Got:** `{'content': 'No pude analizar la imagen: invalid params, invalid image URL'}`
**Root cause hypothesis:** The MiniMax VLM API (`/v1/coding_plan/vlm`) does not accept Wikimedia CDN URLs directly. The image URL `https://upload.wikimedia.org/...` may be rejected due to: (1) the VLM API requires the URL to be a direct image file, not a Wikimedia thumbnail redirect, or (2) the API only accepts certain whitelisted domains. The `direct_understand_image` function passes the URL as-is to `image_url` field without any transformation or validation.

### T09 — n8n API connectivity
**Expected:** Returns list of workflows
**Got:** `httpx.HTTPStatusError: Client error '406 Not Acceptable' for url 'https://n8n.vag131999.cloud/mcp-server/http'`
**Root cause hypothesis:** The n8n MCP server HTTP endpoint returns 406 when the JSON-RPC `tools/list` request is sent with the Instance-level MCP token (audience: `mcp-server-api`). The token may be valid for the n8n REST API but not accepted by the MCP server endpoint, or the MCP endpoint expects a different authentication mechanism. This is a known limitation — the n8n MCP server has separate auth from the REST API.
