## Problema Reportado
El bot respondía "I encountered an error" cuando Fernando pedía las actividades vencidas de CRM en Odoo. El error ocurría silenciosamente sin aparecer en logs.

**Conversación:**
```
[👤 USER] dame las actividades vencidas de CRM
[🤖 BOT] I encountered an error processing your request.
[👤 USER] corrígelo
[🤖 BOT] 📋 Tarea enviada al orquestador...
```

## Causa Raíz
El endpoint `/overdue_activities` estaba completamente implementado en el microservicio FastAPI, pero la herramienta `get_overdue_activities` NO estaba registrada en los diccionarios que el supervisor loop usa para decidir qué herramientas exponer a MiniMax.

**Detalle técnico:**
- **Archivo:** `/app/main.py` (microservicio odoo-python-agent)
- **Función:** `TOOL_DEFINITIONS` + `TOOL_ENDPOINTS`
- **Mecanismo:** El supervisor loop (`run_supervisor_loop`) pasa `TOOL_DEFINITIONS` a MiniMax para que el agente pueda seleccionar herramientas. Si una herramienta no está en esa lista, MiniMax literalmente no puede verla — el tool call nunca se invoca y el flujo falla silenciosamente. El error NO llegaba a los logs porque `execute_tool_call` en brain.py capturaba la excepción y la devolvía como string a MiniMax, que luego la surfacaba como "I encountered an error".

---

## Qué se Corrigió
1. **`get_overdue_activities` agregado a `TOOL_DEFINITIONS`** — el supervisor loop ahora puede ver la herramienta y MiniMax puede seleccionarla para responder "dame las actividades vencidas"
2. **`get_overdue_activities` agregado a `TOOL_ENDPOINTS`** — ahora `call_tool()` puede rutear correctamente hacia `POST /overdue_activities`
3. **Endpoint path de odoo_bridge.py corregido** — de `/get_overdue_activities` (incorrecto) a `/overdue_activities` (correcto)
4. **Validación de startup añadida** — `_validate_tool_registrations()` registra todas las herramientas al iniciar el contenedor, facilitando diagnóstico inmediato

---

## Mejoras Estructurales Añadidas
- **Prevención de recurrencia:** `_validate_tool_registrations()` itera `TOOL_DEFINITIONS` y `TOOL_ENDPOINTS` en cada startup. Si falta alguna herramienta, el log muestra `TOOL_REG_MISSING: 'nombre' in TOOL_DEFINITIONS but no TOOL_ENDPOINTS entry` antes de que cualquier usuario reporte el fallo.
- **Capa de validación:** 5 tests en `test_odoo_tool_registration.py` que verifican consistencia bidireccional entre `TOOL_DEFINITIONS` y `TOOL_ENDPOINTS`
- **Observabilidad:** Logs estructurados al startup mostrando `Tool registration OK. Registered 11 tools: [...]`
- **Sin nuevos errores:** El fix no introdujo errores nuevos en los logs de minimax-agent
