## 3. Causa Raíz

### Versión accesible para Fernando
El archivo `agent/self_improve.py` contiene una función llamada `execute_self_fix()` que se ejecuta cuando el bot recibe un comando "corrígelo". Dentro de esa función hay un bloque de respaldo (fallback) para leer logs de auditoría que fue añadido en una sesión anterior. Ese bloque contenía llamadas a `logger.warning()` en tres líneas (510, 518 y 536), pero el módulo nunca había importado ni definido la variable `logger`.

Cuando el bot detectó errores en el sistema y quiso ejecutar el auto-fix, el bloque de respaldo se activó, llamó a `logger.warning()` y el Python lanza `NameError: name 'logger' is not defined` porque `logger` nunca existía.

### Detalle técnico
- **Archivo:** `/root/minimax-agent/agent/self_improve.py`
- **Función:** `execute_self_fix()` (líneas 510, 518, 536)
- **Mecanismo:** El módulo tenía `from agent.audit_logger import today_summary, today_events` que importa funciones, pero `audit_logger.py` define `_logger` (con guion bajo) no `logger`. El bloque fallback usaba `logger.warning()` sin haber importado `logging` ni definido ningún `logger` en el scope del módulo.
- **Condición de activación:** Cuando `today_summary().get("error", 0) > 0` pero `today_events("tool_call")` devuelve una lista vacía — el fallback se ejecuta y触碰 el `logger.warning()` no definido.
- **Confidence:** 1.0 (evidencia confirmada por traceback + análisis de código)

---

## 4. Qué se Corrigió

Se añadieron las siguientes modificaciones a `self_improve.py`:

1. **`import logging as _logging`** — Added at module level after existing imports
2. **`logger = _logging.getLogger("audit")`** — Module-level binding using the same "audit" logger used by `audit_logger.py`
3. **`_assert_self_improve_logger_bound()`** — New startup assertion function that verifies `logger` is defined and `logger.warning` is callable
4. **Module-level call to `_assert_self_improve_logger_bound()`** — Runs at import time, fails fast if logger is ever removed again

Además, el log de startup `"[startup] LOGGER_BINDING_ASSERTION_PASSED — logger.warning callable in self_improve module"` confirma que la validación se ejecuta correctamente cuando el contenedor inicia.

## 5. Mejoras Estructurales Añadidas

### Prevención de recurrencia
La causa raíz de este bug fue un error de importación dejado por código anterior. El bloque fallback (líneas 507-536) fue añadido en una sesión previa con `logger.warning()` pero sin asegurar que `logger` estuviera importado. La corrección incluye:

- **`_assert_self_improve_logger_bound()`** — Si en el futuro alguien elimina o renombra la línea `logger = _logging.getLogger("audit")`, el bot fallará inmediatamente al iniciar con un `AssertionError` claro, en lugar de crashear silenciosamente cuando un usuario active el comando "corrígelo".

- El archivo de tests `tests/test_self_improve_logger_binding.py` con 5 pruebas valida:

  - `logger` es instancia de `logging.Logger`
  - `logger.warning` es callable
  - La función de assert pasa con binding válido
  - El camino exacto del bug original ahora funciona sin `NameError`
