## 3. Causa Raíz

### Versión Simple (para Fernando)
El problema ocurrió porque el pipeline intentó ejecutar dos agentes simultáneamente (Agent 1 y Agent 2 al mismo tiempo) sin asegurarse de que Agent 1 terminara primero. Cuando Agent 2 empezó antes de que Agent 1 terminara, no tenía los datos necesarios para funcionar correctamente. Esto produjo un "prompt" roto lleno de texto como `{filename}`, `{TRIGGER_PREVIEW}`, `{now}` en lugar de valores reales. Como resultado, Agent 2 no pudo hacer su trabajo y todo el pipeline se detuvo.

### Versión Técnica Completa

**Archivo:** `/root/minimax-agent/pipeline/orchestrator.py`
**Función:** `generate_prompt()` → plantilla f-string en `agents/agent_02_root_cause.py`
**Líneas:** 178-467 (función completa), específicamente 192-209 (sustitución de variables)
**Mecanismo:** En modo paralelo (`asyncio.gather()` en orchestrator.py:745), Agent 2 inicia antes de que Agent 1 termine de escribir `01_failure_trajectory.md`. Cuando `_get_trajectory_content()` retorna string vacío y `_parse_candidates("")` retorna lista vacía, TODAS las variables de plantilla dependientes se convierten en strings vacíos. El template f-string se renderiza con estos valores vacíos, produciendo prompts rotos con literales de placeholder como `{filename}`, `{TRIGGER_PREVIEW}`, `{now}`. El prompt roto se envía a Claude, que no puede producir análisis válido de causa raíz. La validación de `run_agent()` falla porque el contenido requerido "Root Cause (COMMITTED" está ausente del archivo de salida inexistente.

**Evidencia:**
1. `.prompt_temp.md` (9892 bytes, timestamp 03:04:02) contiene variables de plantilla literales sin sustituir — demostrado por `{filename}`, `{function_name}}`, `{TRIGGER_PREVIEW}`, `{now}` en el contenido del prompt
2. El directorio de sesión tiene `01_failure_trajectory.md` (7154 bytes, timestamp 03:04:01) pero NO `02_root_cause.md` — Agent 2 nunca escribió el archivo requerido
3. Agent 1 terminó en ~14 segundos, pero en modo paralelo Agent 2 no esperó
4. `asyncio.gather()` en orchestrator.py:745 ejecuta ambos agentes simultáneamente sin coordinación
5. Recurrencia de 18 veces con 0.0% de tasa de éxito histórico confirma que es un fallo sistémico de infraestructura del pipeline

**Commit que introdujo el problema:** `4e88ceb` — "perf: parallelize Agent 1+2 with asyncio.gather — Agent 2 now independent of Agent 1 output, saves 10-15min per pipeline"
