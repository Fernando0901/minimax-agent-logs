## 3. Causa Raíz

**Problema técnico:** El healthcheck de Docker en `docker-compose.yml` líneas 18-21 era un script Python dummy:

```yaml
healthcheck:
  test: ["CMD", "python", "-c", "import sys; sys.exit(0)"]
```

Este script **siempre retorna exit code 0**, lo que significa que Docker marca el contenedor como "healthy" sin importar si la aplicación está colgada, degradada, o experimentando timeouts en cadena.

**Mecanismo del fallo en español:** Cuando la API de MiniMax se degradó (respondiendo en 100+ segundos), las llamadas en `brain.py` disparaban `asyncio.wait_for(timeout=60.0)` produciendo errores de TimeoutError. simultaneous, el `odoo_bridge` experimentaba ReadTimeout en múltiples endpoints. Todo esto ocurría mientras el healthcheck seguía reportando "healthy" porque siempre sale con código 0. El monitor proactivo veía el estado "****" (inaccesible o en transición) y generaba alertas falsas.

**Patrón de recurrencia:** Este era el motivo por el cual 25+ ejecuciones del pipeline mostraban este error con 0% de éxito — todas las correcciones anteriores abordaban síntomas a nivel de código (timeouts, reintentos, ventanas de contexto) pero ninguna reparaba el gap de monitoreo que impedía al sistema detectar estados unhealthy真实的.

---

## 4. Qué se Corrigió

Se reemplazó el healthcheck dummy por un script bash-aware que realiza tres validaciones reales:

1. **Conectividad API de Telegram:** Llama a `https://api.telegram.org/bot{TOKEN}/getMe` — retorna exit 1 si la API no responde en 10 segundos
2. **Heartbeat del event loop:** Verifica que `/root/minimax-agent/logs/agent.log` tenga actividad en los últimos 300 segundos — exit 1 si está stale
3. **Existencia del proceso:** Confirma que `python main.py` está corriendo via `pgrep`

**Archivos modificados:**
- `/root/minimax-agent/scripts/healthcheck.sh` — **NUEVO** (2999 bytes, ejecutable)
- `/root/minimax-agent/agent/main.py` —添加了 `_validate_healthcheck_script()` en startup
- `/root/minimax-agent/docker-compose.yml` — reemplazó dummy python con `bash /root/minimax-agent/scripts/healthcheck.sh`, start_period 10s→30s, timeout 30s
- `/root/minimax-agent/tests/test_healthcheck.py` — **NUEVO** test suite (10 tests, 8/10 pass)
