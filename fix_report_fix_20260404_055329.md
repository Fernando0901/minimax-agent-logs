## Fix Aplicado

**Problema:** El bot mostraba `{"tool_call":{"name":...}}` como texto en lugar de ejecutar las herramientas.

**Causa raíz:** El modelo MiniMax M2.5 no usaba el mecanismo nativo de function calling de la API. En cambio, outputs el tool_call como texto JSON siguiendo las instrucciones del system prompt que incluía un ejemplo incorrecto.

**Cambios en `brain.py`:**

1. **Workaround para tool calls en texto** — Si el modelo outputs un tool_call como texto en lugar de usar el campo `tool_calls` de la API, el código ahora:
   - Detecta el patrón `{"tool_call": ...}` en el texto
   - Extrae y parsea el JSON
   - Ejecuta la herramienta
   - Continúa el loop para obtener la respuesta final

2. **System prompt corregido** — Eliminé el ejemplo `EXAMPLE_TOOL_CALL` y la instrucción de outputs JSON. Ahora dice: "When you need to use a tool, the system will handle tool calls automatically."
