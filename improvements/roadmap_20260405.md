### R01 — understand_image fails with Wikimedia URLs [P1]
**Broken test:** T08
**Root cause:** The MiniMax VLM API (`/v1/coding_plan/vlm`) rejects Wikimedia CDN URLs (`https://upload.wikimedia.org/...`). The `direct_understand_image` function in `brain.py` passes URLs directly without validation or transformation. Wikimedia thumbnail URLs are HTTP redirects that the VLM API cannot follow.
**Files to modify:** `agent/brain.py` (direct_understand_image function, lines 267-308)
**Exact fix:**
In `direct_understand_image`, before sending to the VLM API, detect Wikimedia/westricted URLs and fetch the image data locally first:

```python
async def direct_understand_image(prompt: str, image_source: str) -> dict:
    import base64
    api_key = os.getenv("MINIMAX_API_KEY")
    api_host = os.getenv("MINIMAX_API_HOST", "https://api.minimax.io")

    headers = {
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json"
    }

    # If local file path, convert to base64 data URI
    if image_source.startswith("/") or image_source.startswith("./"):
        if os.path.exists(image_source):
            ext = os.path.splitext(image_source)[1].lower()
            mime = "image/jpeg" if ext in [".jpg", ".jpeg"] else "image/png" if ext == ".png" else "image/webp"
            with open(image_source, "rb") as f:
                b64 = base64.b64encode(f.read()).decode()
            image_source = f"data:{mime};base64,{b64}"
        else:
            return {"content": f"Archivo no encontrado: {image_source}"}
    else:
        # For HTTP URLs (especially Wikimedia and potentially blocked domains),
        # fetch the image data and convert to base64 to ensure API compatibility
        try:
            async with httpx.AsyncClient(timeout=30.0, follow_redirects=True) as img_client:
                img_resp = await img_client.get(image_source)
                img_resp.raise_for_status()
                content_type = img_resp.headers.get("content-type", "image/png")
                b64 = base64.b64encode(img_resp.content).decode()
                image_source = f"data:{content_type};base64,{b64}"
        except Exception as img_err:
            return {"content": f"No pude descargar la imagen: {str(img_err)}"}

    payload = {
        "prompt": prompt,
        "image_url": image_source
    }

    async with httpx.AsyncClient(timeout=60.0) as client:
        resp = await client.post(
            f"{api_host}/v1/coding_plan/vlm",
            headers=headers,
            json=payload
        )
        resp.raise_for_status()
        data = resp.json()

    content = data.get("content", "")
    if not content:
        base_resp = data.get("base_resp", {})
        status_msg = base_resp.get("status_msg", "Error desconhecido")
        return {"content": f"No pude analizar la imagen: {status_msg}"}
    return {"content": content}
```

**Verify with:** `python3 -c "import asyncio; from agent.brain import direct_understand_image; print(asyncio.run(direct_understand_image('What is in this image?', 'https://upload.wikimedia.org/wikipedia/commons/thumb/a/a7/Camponotus_flavomarginatus_ant.jpg/120px-Camponotus_flavomarginatus_ant.jpg')))"`
**Risk:** Low — only affects external HTTP URLs; local files still work as before. Adding `follow_redirects=True` is safe.

### R02 — n8n MCP endpoint returns 406 Not Acceptable [P1]
**Broken test:** T09
**Root cause:** The n8n MCP server HTTP endpoint (`/mcp-server/http`) returns 406 when the Instance-level MCP token (audience: `mcp-server-api`) is used. The token is valid for the n8n REST API but not accepted by the MCP JSON-RPC endpoint. The MCP server likely requires an MCP-specific token or different auth mechanism.
**Files to modify:** `agent/brain.py` (n8n_list_workflows and n8n_trigger_workflow functions, lines 212-264)
**Alternative approach:** Keep current n8n functions as-is but add a fallback using the n8n REST API directly (`/api/v1/workflows`) which does accept the current token.
**Exact fix:**
Replace the MCP JSON-RPC calls with direct REST API calls:

```python
async def n8n_list_workflows() -> dict:
    """List all n8n workflows via direct REST API (bypasses MCP endpoint)."""
    n8n_url = os.getenv("N8N_MCP_URL", "https://n8n.vag131999.cloud")
    n8n_token = os.getenv("N8N_MCP_TOKEN", "")

    headers = {
        "Authorization": f"Bearer {n8n_token}",
        "Content-Type": "application/json"
    }

    # Use n8n REST API directly — works with the same token
    async with httpx.AsyncClient(timeout=30.0) as client:
        resp = await client.get(
            f"{n8n_url}/api/v1/workflows",
            headers=headers
        )
        resp.raise_for_status()
        data = resp.json()
        workflows = [
            {"name": w.get("name", ""), "id": w.get("id", ""), "active": w.get("active", False)}
            for w in data.get("data", [])
        ]
        return {"workflows": workflows}


async def n8n_trigger_workflow(workflow_id: str, payload: dict = None) -> dict:
    """Trigger an n8n workflow by ID via REST API execution endpoint."""
    n8n_url = os.getenv("N8N_MCP_URL", "https://n8n.vag131999.cloud")
    n8n_token = os.getenv("N8N_MCP_TOKEN", "")

    headers = {
        "Authorization": f"Bearer {n8n_token}",
        "Content-Type": "application/json"
    }

    # Use n8n REST API to trigger workflow execution
    async with httpx.AsyncClient(timeout=60.0) as client:
        resp = await client.post(
            f"{n8n_url}/api/v1/workflows/{workflow_id}/execute",
            headers=headers,
            json=payload or {}
        )
        resp.raise_for_status()
        return resp.json()
```

Note: The `/execute` endpoint may not exist on all n8n versions. If 404, fallback to using n8n's webhook trigger URL if one is configured, or document that workflow triggering requires manual trigger.

**Verify with:** `python3 -c "import asyncio; from agent.brain import n8n_list_workflows; print(asyncio.run(n8n_list_workflows()))"`
**Risk:** Medium — changes auth from MCP JSON-RPC to REST API. REST API token permissions must be verified. If the token lacks `workflow:read` scope, this will also fail with 403.
